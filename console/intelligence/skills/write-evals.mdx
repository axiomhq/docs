The Write evals skill turns AI agents into evaluation suite authors for AI capabilities.

Evaluations are the test suite for non-deterministic systems: they measure whether a capability still behaves correctly after every change. You can write offline evals to test your AI capability against a curated collection of inputs with expected outputs (ground truth). Or you can write online evals to score your AI capability's outputs on live production traffic.

For more information, see [Run offline evaluations](/ai-engineering/evaluate/run-evaluations) and [Run online evaluations](/ai-engineering/evaluate/online-evaluations/write-run-evaluations).

- **Eval generation**: Creates `.eval.ts` files with typed scorers, test data, and task functions
- **Scorer patterns**: Exact match, set match, structured output, tool-use validation, LLM-as-judge, and more
- **Flag schemas**: Generates typed configuration with `createAppScope()` for sweeping models, temperatures, and strategies without code changes
- **Online evals**: Reference-free scoring on live production traffic with sampling and trace linking
- **Data design**: Generates test cases covering happy path, adversarial, boundary, and negative categories
- **Templates**: Pre-built eval file templates for common patterns

<Prerequisites />

- Install [Axiom AI SDK](https://www.npmjs.com/package/axiom) by running `npm install axiom`.

## Use the skill

The Write evals skill activates automatically when you ask your AI agent to:

- Write evaluations for an AI feature
- Create scorers for testing AI output
- Set up flag schemas for experiment configuration
- Generate test data for AI capabilities

Example prompts:

- "Write evals for the support agent's message categorization function"
- "Add adversarial test cases to the existing eval suite"
- "Set up flag schemas to compare GPT-4o and GPT-5 across all evals"
- "Create an online eval to monitor response quality in production"

## How it works

The Write evals skill follows a structured workflow:

1. **Understand the feature**: The agent reads your AI function's code, traces inputs and outputs, identifies the model call, and checks for existing evals.
1. **Determine eval type**: Based on the output type (classification, free-form text, structured object, tool calls, retrieval results), the agent selects the appropriate eval pattern and scorers.
1. **Generate eval file**: The agent creates a `.eval.ts` file colocated with the source, imports the actual function, and wires up typed scorers.
1. **Design test data**: The agent generates test cases covering happy path, adversarial, boundary, and negative categories.
1. **Validate and run**: The agent validates the eval structure and runs it locally.

## Eval types

The Write evals skill supports the following eval types:

| Output type | Eval type | Scorer pattern |
| --- | --- | --- |
| String category or label | Classification | Exact match |
| Free-form text | Text quality | Contains keywords or LLM-as-judge |
| Array of items | Retrieval | Set match |
| Structured object | Structured output | Field-by-field match |
| Agent result with tool calls | Tool use | Tool name presence |

## Eval templates

The Write evals skill includes pre-built templates:

| Template | Use case |
| --- | --- |
| Minimal | Simplest eval with exact match scorer |
| Classification | Category classification with adversarial cases |
| Retrieval | RAG and document retrieval with set matching |
| Structured output | Complex object validation with field-by-field scoring |
| Tool use | Agent tool usage validation |
