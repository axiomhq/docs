## Determine organization ID

1. Click <Icon icon="gear" iconType="solid"/> **Settings > General**.

## Set default dataset retention period

The default dataset retention period determines how long Axiom stores data in datasets that don’t have their own custom retention period. Datasets without a custom retention period inherit this value. You can only set the default dataset retention period on the Axiom Cloud plan.

<Warning>
Changing the default dataset retention period affects **existing datasets**, not only datasets you create later. When you lower the default, Axiom immediately deletes all data older than the new period in every dataset that inherits the default. This process can’t be undone.

Before you lower the default, set a custom retention period on each dataset whose data you want to keep. For more information, see [Specify data retention period](/reference/datasets#specify-data-retention-period).
</Warning>

The default dataset retention period also determines the maximum query window of your organization. This means the following:

- You can’t query data older than the default dataset retention period, even if an individual dataset has a longer custom retention period.
- If a query reaches further back than the default dataset retention period, Axiom shortens the query to the maximum query window instead of returning an error. As a result, the query returns fewer results than you expect.
- If the entire time range of a query is older than the default dataset retention period, the query returns an error.

To store and query data for longer, increase the default dataset retention period. Increasing the default doesn’t delete any data.

To set the default dataset retention period, follow these steps:

1. Click <Icon icon="gear" iconType="solid"/> **Settings > General**.
1. Click <Icon icon="pen-to-square" iconType="regular"/> next to the **Default dataset retention** section.
1. Specify the default dataset retention period.
1. Click **Submit**.

## Delete organization

<Warning>
This is a destructive action. After you delete your organization, you lose access to all data within that org.
</Warning>

To delete your organization:

1. Back up your data. You aren’t able to access the data after deleting the org.
1. Click <Icon icon="gear" iconType="solid"/> **Settings > General**
1. Click **Delete organization**.
