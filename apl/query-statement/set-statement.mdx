The `set` statement configures query options. Options enabled with the `set` statement apply only for the duration of the query.

The `set` statement affects how your query is processed and determines the returned results.

## Syntax 

```kusto
set OptionName=OptionValue
```

## Strict types

The `stricttypes` query option requires exact data type declarations in your query. If the exact type isn't specified, a **QueryFailed** error is thrown.

## Example 

```kusto
set stricttypes; 
['Dataset'] 
| where number == 5
```
