## Introduction

The `project-reorder` operator in APL allows you to rearrange the columns of a dataset without modifying the underlying data. This operator is useful when you need to control the display order of columns in query results, making your data easier to read and analyze. It can be especially helpful when working with large datasets where column ordering impacts the clarity of the output.

Use `project-reorder` when you want to emphasize specific fields by adjusting their order in the result set without changing their values or structure.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk SPL, you use the `table` command to reorder columns, which works similarly to how `project-reorder` functions in APL.

<CodeGroup>
```splunk Splunk example
| table columnA, columnB, columnC
```

```kusto APL equivalent
['dataset.name']
| project-reorder columnA, columnB, columnC
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In ANSI SQL, the order of columns in a `SELECT` statement determines their arrangement in the output. In APL, `project-reorder` provides more explicit control over the column order without requiring a full `SELECT` clause.

<CodeGroup>
```sql SQL example
SELECT columnA, columnB, columnC FROM dataset;
```

```kusto APL equivalent
| project-reorder columnA, columnB, columnC
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
| project-reorder column1, column2, ..., columnN
```

### Parameters

- `column1, column2, ..., columnN`: The names of the columns in the order you want them to appear in the result set.

### Returns

The reordered dataset with columns displayed in the specified order.

## Use case examples

<Tabs>
<Tab title="Log analysis">

In this example, you reorder HTTP log fields to prioritize the most relevant ones for log analysis.

**Query**

```kusto
['sample-http-logs']
| project-reorder _time, method, status, uri, req_duration_ms, ['geo.city'], ['geo.country']
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/explorer?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20project-reorder%20_time%2C%20method%2C%20status%2C%20uri%2C%20req_duration_ms%2C%20%5B%27geo.city%27%5D%2C%20%5B%27geo.country%27%5D%22%7D)

**Output**

| _time               | method | status | uri               | req_duration_ms | geo.city   | geo.country |
|---------------------|--------|--------|-------------------|-----------------|------------|-------------|
| 2024-10-17 12:34:56 | GET    | 200    | /home              | 120             | New York   | USA         |
| 2024-10-17 12:35:01 | POST   | 404    | /api/v1/resource   | 250             | Berlin     | Germany     |

This query rearranges the columns for clarity, placing the most crucial fields (`_time`, `method`, `status`) at the front for easier analysis.

## List of related operators

- [**`project`**](/apl/tabular-operators/project-operator): Use the `project` operator to select and rename columns without changing their order.
- [**`extend`**](/apl/tabular-operators/extend-operator): `extend` adds new calculated columns while keeping the original ones in place.
- [**`summarize`**](/apl/tabular-operators/summarize-operator): Use `summarize` to perform aggregations on columns, which can then be reordered using `project-reorder`.
