Use the `field_metadata` function in APL to retrieve metadata about a specific field in your dataset. This function returns information such as field type, units, and other schema-related properties. This is particularly useful when working with metrics data or when you need to programmatically access field characteristics.

You use `field_metadata` when building dynamic queries that adapt based on field properties, performing schema introspection, or working with metrics that have associated units and intervals. This function is especially valuable in automated analytics, data validation, and building adaptive dashboards.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk SPL, you use commands like `metadata` or `fieldsummary` to retrieve field information. APL's `field_metadata` provides direct access to field metadata as part of your query pipeline.

<CodeGroup>
```sql Splunk example
| metadata type=fields index=main
```

```kusto APL equivalent
['sample-http-logs']
| extend req_duration_metadata = field_metadata('req_duration_ms')
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In ANSI SQL, you query the information schema to retrieve column metadata. APL's `field_metadata` brings this capability into your data queries, allowing you to use metadata dynamically within your analysis.

<CodeGroup>
```sql SQL example
SELECT column_name, data_type, column_default
FROM information_schema.columns
WHERE table_name = 'logs' AND column_name = 'duration'
```

```kusto APL equivalent
['sample-http-logs']
| extend metadata = field_metadata('req_duration_ms')
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
field_metadata(field_name)
field_metadata(field_name, dataset_name)
```

### Parameters

| Name | Type | Description |
| ---- | ---- | ----------- |
| `field_name` | `string` | The name of the field for which you want to retrieve metadata. Must be a constant string. |
| `dataset_name` | `string` | (Optional) The name of the dataset containing the field. If not specified, uses the current dataset context. |

### Returns

A `dynamic` object containing metadata about the specified field, including properties like:
- `unit` - The unit of measurement for the field (if applicable)
- `metrics_interval` - The interval for metrics data (if applicable)
- Other schema-related properties depending on the field type

Returns `null` if the field doesn't exist or has no metadata.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `field_metadata` to dynamically access field properties for adaptive query logic in HTTP request analysis.

**Query**

```kusto
['sample-http-logs']
| extend duration_meta = field_metadata('req_duration_ms')
| project _time, req_duration_ms, duration_meta
| take 5
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B'sample-http-logs'%5D%20%7C%20extend%20duration_meta%20%3D%20field_metadata('req_duration_ms')%20%7C%20project%20_time%2C%20req_duration_ms%2C%20duration_meta%20%7C%20take%205%22%7D)

**Output**

| _time              | req_duration_ms | duration_meta            |
| ------------------ | --------------- | ------------------------ |
| 2025-05-26 08:15:30 | 156            | \{...\}                   |
| 2025-05-26 08:16:45 | 203            | \{...\}                   |

This query retrieves metadata for the request duration field, which you can use to understand field characteristics programmatically.

</Tab>
<Tab title="OpenTelemetry traces">

Use `field_metadata` to access span duration metadata for building adaptive trace analysis queries.

**Query**

```kusto
['otel-demo-traces']
| extend duration_meta = field_metadata('duration')
| project _time, duration, duration_meta, ['service.name']
| take 5
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B'otel-demo-traces'%5D%20%7C%20extend%20duration_meta%20%3D%20field_metadata('duration')%20%7C%20project%20_time%2C%20duration%2C%20duration_meta%2C%20%5B'service.name'%5D%20%7C%20take%205%22%7D)

**Output**

| 2025-05-26 10:20:15 | 0.125s   | \{...\}        | frontend          |

This query accesses duration field metadata, useful for understanding time units and building normalized trace visualizations.

</Tab>
<Tab title="Security logs">

Use `field_metadata` to understand field properties for building adaptive security monitoring queries.

**Query**

```kusto
['sample-http-logs']
| extend status_meta = field_metadata('status')
| where status != '200'
| project _time, status, uri, status_meta
| take 5
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B'sample-http-logs'%5D%20%7C%20extend%20status_meta%20%3D%20field_metadata('status')%20%7C%20where%20status%20!%3D%20'200'%20%7C%20project%20_time%2C%20status%2C%20uri%2C%20status_meta%20%7C%20take%205%22%7D)

**Output**

| _time              | status | uri          | status_meta |
| ------------------ | ------ | ------------ | ----------- |
| 2025-05-26 11:30:00 | 404    | /api/missing | \{...\}      |
| 2025-05-26 11:31:15 | 403    | /admin       | \{...\}      |

This query retrieves metadata for security-relevant fields, helping you build adaptive security analytics.
