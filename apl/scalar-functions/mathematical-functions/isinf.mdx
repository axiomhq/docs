Use the `isinf` function in APL to check whether a numeric value is positive or negative infinity. The function returns `true` only when the value is ±∞ and `false` for all finite values and NaN.

`isinf` is useful for detecting overflow conditions or division-by-zero results in computed columns. Unlike `isfinite`, which also catches NaN values, `isinf` specifically targets infinite results, allowing you to distinguish overflow from undefined computations.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

Splunk SPL doesn't have a direct `isinf()` function. You typically check whether a value equals a specific infinity representation, which isn't directly available in most SPL expressions.

<CodeGroup>
```sql Splunk example
| eval is_inf = if(value = 'Infinity' OR value = '-Infinity', 1, 0)
````

```kusto APL equivalent
['sample-http-logs']
| extend is_inf = isinf(value)
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

Standard SQL does not define an `ISINF()` function. Most SQL databases raise an error on division by zero rather than producing infinity, so there is no direct equivalent.

<CodeGroup>
```sql SQL example
-- Division by zero raises an error in most SQL dialects; no direct ISINF() equivalent
SELECT CASE WHEN value > 1e308 OR value < -1e308 THEN 1 ELSE 0 END AS is_inf FROM logs
```

```kusto APL equivalent
['sample-http-logs']
| extend is_inf = isinf(value)
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
isinf(x)
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `x` | real | Yes | The numeric value to check. |

### Returns

`true` if `x` is positive or negative infinity. `false` for finite values and NaN.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `isinf` to detect rows where a computed ratio produces infinity, which happens when dividing a non-zero value by zero.

**Query**

```kusto
['sample-http-logs']
| extend ratio = req_duration_ms / 0.0
| extend is_inf = isinf(ratio)
| project _time, id, req_duration_ms, ratio, is_inf
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20extend%20ratio%20%3D%20req_duration_ms%20%2F%200.0%20%7C%20extend%20is_inf%20%3D%20isinf%28ratio%29%20%7C%20project%20_time%2C%20id%2C%20req_duration_ms%2C%20ratio%2C%20is_inf%22%7D)

**Output**

| _time | id | req_duration_ms | ratio | is_inf |
|---|---|---|---|---|
| 2024-11-14 10:00:00 | user-1 | 120.0 | inf | true |
| 2024-11-14 10:01:00 | user-2 | 0.0 | nan | false |
| 2024-11-14 10:02:00 | user-3 | 80.0 | inf | true |

Non-zero durations divided by zero produce `inf`, which `isinf` flags as `true`. A zero duration divided by zero produces `nan`, which `isinf` returns `false` for. Use `isnan` to handle that case separately.

</Tab>
<Tab title="OpenTelemetry traces">

Use `isinf` to identify spans where a derived metric calculation overflows before aggregating across services.

**Query**

```kusto
['otel-demo-traces']
| extend duration_ms = duration / 1ms
| extend ratio = duration_ms / 0.0
| extend is_inf = isinf(ratio)
| project _time, span_id, ['service.name'], duration_ms, is_inf
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27otel-demo-traces%27%5D%20%7C%20extend%20duration_ms%20%3D%20duration%20%2F%201ms%20%7C%20extend%20ratio%20%3D%20duration_ms%20%2F%200.0%20%7C%20extend%20is_inf%20%3D%20isinf%28ratio%29%20%7C%20project%20_time%2C%20span_id%2C%20%5B%27service.name%27%5D%2C%20duration_ms%2C%20is_inf%22%7D)

**Output**

| _time | span_id | service.name | duration_ms | is_inf |
|---|---|---|---|---|
| 2024-11-14 10:00:00 | span-abc | checkout | 350.0 | true |
| 2024-11-14 10:01:00 | span-def | cart | 0.0 | false |

Spans with `is_inf = true` have a non-zero duration that produced infinity from the calculation. Filter these out before computing any averages or percentiles.

</Tab>
<Tab title="Security logs">

Use `isinf` to clean computed fields in error-event analysis, ensuring that infinity values from faulty calculations do not distort security metrics.

**Query**

```kusto
['sample-http-logs']
| where status == '500'
| extend ratio = req_duration_ms / 0.0
| extend is_inf = isinf(ratio)
| project _time, id, status, req_duration_ms, is_inf
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20status%20%3D%3D%20%27500%27%20%7C%20extend%20ratio%20%3D%20req_duration_ms%20%2F%200.0%20%7C%20extend%20is_inf%20%3D%20isinf%28ratio%29%20%7C%20project%20_time%2C%20id%2C%20status%2C%20req_duration_ms%2C%20is_inf%22%7D)

**Output**

| _time | id | status | req_duration_ms | is_inf |
|---|---|---|---|---|
| 2024-11-14 10:00:00 | user-9 | 500 | 120.0 | true |
| 2024-11-14 10:01:00 | user-5 | 500 | 0.0 | false |

Before aggregating error metrics, add `| where not(is_inf)` to exclude rows with overflow results and keep your aggregations reliable.

## List of related functions

- [isfinite](/apl/scalar-functions/mathematical-functions/isfinite): Returns `true` for values that are neither infinite nor NaN. Use it for a broader check covering all invalid float states.
- [isnan](/apl/scalar-functions/mathematical-functions/isnan): Returns `true` only for NaN values. Use it alongside `isinf` to detect the other class of invalid float.
- [isint](/apl/scalar-functions/mathematical-functions/isint): Returns `true` for integer values. Use it when you need to validate that a value is a whole number.
- [not](/apl/scalar-functions/mathematical-functions/not): Reverses a boolean value. Use `not(isinf(x))` as a shorthand for `isfinite(x)` when NaN values are not a concern.
- [log](/apl/scalar-functions/mathematical-functions/log): Returns the natural logarithm. Compute `log` on potentially zero or negative values and use `isinf` to detect the resulting `-inf`.
