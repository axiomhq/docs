Use the `sin` function in APL to compute the sine of an angle expressed in radians. The function accepts any real number and returns a value in the range [-1, 1].

`sin` is most commonly used in observability to encode time-of-day or other periodic signals as cyclic coordinates. When combined with `cos`, it produces a two-dimensional representation of periodic phenomena that preserves circular distance. For example, you can encode hour-of-day cyclically so that hour 23 and hour 0 are treated as adjacent.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk SPL, `sin()` takes an angle in radians and returns the sine, just like APL.

<CodeGroup>
```sql Splunk example
| eval sin_val = sin(angle_rad)
````

```kusto APL equivalent
['sample-http-logs']
| extend sin_val = sin(angle_rad)
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In ANSI SQL, `SIN()` is a standard function with identical semantics.

<CodeGroup>
```sql SQL example
SELECT SIN(angle_rad) AS sin_val FROM logs
```

```kusto APL equivalent
['sample-http-logs']
| extend sin_val = sin(angle_rad)
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
sin(x)
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `x` | real | Yes | The angle in radians. |

### Returns

The sine of `x`, a real number in the range [-1, 1].

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `sin` to encode the hour of day as a cyclic feature. Together with `cos`, this produces a unit-circle representation of time that avoids the discontinuity between hour 23 and hour 0.

**Query**

```kusto
['sample-http-logs']
| extend hour_angle = (hourofday(_time) * 2 * pi()) / 24
| extend sin_hour = sin(hour_angle)
| project _time, id, req_duration_ms, sin_hour
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20extend%20hour_angle%20%3D%20%28hourofday%28_time%29%20%2A%202%20%2A%20pi%28%29%29%20%2F%2024%20%7C%20extend%20sin_hour%20%3D%20sin%28hour_angle%29%20%7C%20project%20_time%2C%20id%2C%20req_duration_ms%2C%20sin_hour%22%7D)

**Output**

| _time | id | req_duration_ms | sin_hour |
|---|---|---|---|
| 2024-11-14 00:00:00 | user-1 | 120.0 | 0.0000 |
| 2024-11-14 06:00:00 | user-2 | 80.0 | 1.0000 |
| 2024-11-14 12:00:00 | user-3 | 200.0 | 0.0000 |
| 2024-11-14 18:00:00 | user-4 | 150.0 | -1.0000 |

`sin_hour` is 0 at midnight and noon, reaches +1 at 6 AM, and -1 at 6 PM. Use this coordinate alongside `cos_hour` for complete cyclic time encoding.

</Tab>
<Tab title="OpenTelemetry traces">

Use `sin` to add a cyclic time feature to trace data, enabling time-of-day correlation analysis across services.

**Query**

```kusto
['otel-demo-traces']
| extend hour_angle = (hourofday(_time) * 2 * pi()) / 24
| extend sin_hour = sin(hour_angle)
| project _time, span_id, ['service.name'], duration, sin_hour
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27otel-demo-traces%27%5D%20%7C%20extend%20hour_angle%20%3D%20%28hourofday%28_time%29%20%2A%202%20%2A%20pi%28%29%29%20%2F%2024%20%7C%20extend%20sin_hour%20%3D%20sin%28hour_angle%29%20%7C%20project%20_time%2C%20span_id%2C%20%5B%27service.name%27%5D%2C%20duration%2C%20sin_hour%22%7D)

**Output**

| _time | span_id | service.name | duration | sin_hour |
|---|---|---|---|---|
| 2024-11-14 06:00:00 | span-abc | checkout | 00:00:00.3500000 | 1.0000 |

| 2024-11-14 18:00:00 | span-ghi | frontend | 00:00:00.0800000 | -1.0000 |

The `sin_hour` feature enables you to correlate span duration with position in the daily traffic cycle without the discontinuity of a raw hour value.

</Tab>
<Tab title="Security logs">

Use `sin` to detect whether HTTP 500 errors cluster at specific phases of the day cycle, which can reveal periodic attack patterns.

**Query**

```kusto
['sample-http-logs']
| where status == '500'
| extend hour_angle = (hourofday(_time) * 2 * pi()) / 24
| extend sin_hour = sin(hour_angle)
| project _time, id, status, uri, sin_hour
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20status%20%3D%3D%20%27500%27%20%7C%20extend%20hour_angle%20%3D%20%28hourofday%28_time%29%20%2A%202%20%2A%20pi%28%29%29%20%2F%2024%20%7C%20extend%20sin_hour%20%3D%20sin%28hour_angle%29%20%7C%20project%20_time%2C%20id%2C%20status%2C%20uri%2C%20sin_hour%22%7D)

**Output**

| _time | id | status | uri | sin_hour |
|---|---|---|---|---|
| 2024-11-14 02:00:00 | user-9 | 500 | /api/checkout | 0.5000 |
| 2024-11-14 14:00:00 | user-5 | 500 | /api/cart | -0.8660 |

If error events cluster near the same `sin_hour` value across different days, they likely follow a periodic schedule and may be part of an automated attack pattern.

## List of related functions

- [cos](/apl/scalar-functions/mathematical-functions/cos): Returns the cosine. Use `sin` and `cos` together to produce a two-dimensional cyclic encoding of angles.
- [tan](/apl/scalar-functions/mathematical-functions/tan): Returns the tangent. Use it when you need the ratio of sine to cosine.
- [asin](/apl/scalar-functions/mathematical-functions/asin): Returns the arc sine. Use it as the inverse of `sin`.
- [pi](/apl/scalar-functions/mathematical-functions/pi): Returns the constant π. Use it to construct radian values before calling `sin`.
- [radians](/apl/scalar-functions/mathematical-functions/radians): Converts degrees to radians. Use it to prepare degree inputs before calling `sin`.
