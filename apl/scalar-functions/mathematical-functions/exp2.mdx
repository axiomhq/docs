Use the `exp2` function in APL to compute the base-2 exponential of a value: 2^x. It is the inverse of the base-2 logarithm function `log2`.

`exp2` is useful when you work with data measured on a binary scale, such as memory sizes, network bandwidths, or binary tree depths. You can also use it to reverse a `log2` transformation and recover values on the original scale.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

Splunk SPL does not include a built-in `exp2()` function. You compute it as `pow(2, x)`.

<CodeGroup>
```sql Splunk example
| eval result = pow(2, x)
````

```kusto APL equivalent
['sample-http-logs']
| extend result = exp2(x)
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

Standard SQL does not define `EXP2()`. You compute it using `POWER(2, x)`.

<CodeGroup>
```sql SQL example
SELECT POWER(2, x) AS result FROM logs
```

```kusto APL equivalent
['sample-http-logs']
| extend result = exp2(x)
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
exp2(x)
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `x` | real | Yes | The exponent value. |

### Returns

The base-2 exponential of `x`: 2^x.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `exp2` to recover the geometric mean of request durations after a `log2` transformation, expressing the result in the original millisecond scale.

**Query**

```kusto
['sample-http-logs']
| where req_duration_ms > 0
| summarize avg_log2 = avg(log2(req_duration_ms)) by bin(_time, 1h)
| extend geometric_mean = exp2(avg_log2)
| project _time, avg_log2, geometric_mean
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20req_duration_ms%20%3E%200%20%7C%20summarize%20avg_log2%20%3D%20avg%28log2%28req_duration_ms%29%29%20by%20bin%28_time%2C%201h%29%20%7C%20extend%20geometric_mean%20%3D%20exp2%28avg_log2%29%20%7C%20project%20_time%2C%20avg_log2%2C%20geometric_mean%22%7D)

**Output**

| _time | avg_log2 | geometric_mean |
|---|---|---|
| 2024-11-14 10:00:00 | 6.42 | 85.3 |
| 2024-11-14 11:00:00 | 6.54 | 92.7 |
| 2024-11-14 12:00:00 | 6.29 | 78.1 |

`exp2(avg(log2(x)))` computes the geometric mean in base-2 space. This is the binary equivalent of `exp(avg(log(x)))` and is useful when data spans multiple powers of two.

</Tab>
<Tab title="OpenTelemetry traces">

Use `exp2` to convert log2-transformed span durations back to milliseconds after summarizing.

**Query**

```kusto
['otel-demo-traces']
| extend duration_ms = duration / 1ms
| where duration_ms > 0
| summarize avg_log2_duration = avg(log2(duration_ms)) by ['service.name'], bin(_time, 1h)
| extend geometric_mean = exp2(avg_log2_duration)
| project _time, ['service.name'], avg_log2_duration, geometric_mean
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27otel-demo-traces%27%5D%20%7C%20extend%20duration_ms%20%3D%20duration%20%2F%201ms%20%7C%20where%20duration_ms%20%3E%200%20%7C%20summarize%20avg_log2_duration%20%3D%20avg%28log2%28duration_ms%29%29%20by%20%5B%27service.name%27%5D%2C%20bin%28_time%2C%201h%29%20%7C%20extend%20geometric_mean%20%3D%20exp2%28avg_log2_duration%29%20%7C%20project%20_time%2C%20%5B%27service.name%27%5D%2C%20avg_log2_duration%2C%20geometric_mean%22%7D)

**Output**

|---|---|---|---|
| 2024-11-14 10:00:00 | checkout | 8.13 | 280.5 |

<Tab title="Security logs">

Use `exp2` to recover absolute duration values from log2-summarized error responses per region, making it easy to identify regions with the slowest error processing.

**Query**

```kusto
['sample-http-logs']
| where status == '500' and req_duration_ms > 0
| summarize avg_log2 = avg(log2(req_duration_ms)) by ['geo.country']
| extend avg_duration = exp2(avg_log2)
| project ['geo.country'], avg_log2, avg_duration
| order by avg_duration desc
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20status%20%3D%3D%20%27500%27%20and%20req_duration_ms%20%3E%200%20%7C%20summarize%20avg_log2%20%3D%20avg%28log2%28req_duration_ms%29%29%20by%20%5B%27geo.country%27%5D%20%7C%20extend%20avg_duration%20%3D%20exp2%28avg_log2%29%20%7C%20project%20%5B%27geo.country%27%5D%2C%20avg_log2%2C%20avg_duration%20%7C%20order%20by%20avg_duration%20desc%22%7D)

**Output**

| geo.country | avg_log2 | avg_duration |
|---|---|---|
| BR | 8.95 | 492.7 |
| DE | 7.64 | 200.3 |
| US | 6.99 | 128.0 |

Regions at the top of the list have the highest average error-response durations, which may indicate infrastructure bottlenecks or targeted slow-response patterns.

## List of related functions

- [log2](/apl/scalar-functions/mathematical-functions/log2): Returns the base-2 logarithm. Use it as the inverse of `exp2`.
- [exp](/apl/scalar-functions/mathematical-functions/exp): Returns e^x. Use it for natural-log-scale transformations instead of base-2.
- [exp10](/apl/scalar-functions/mathematical-functions/exp10): Returns 10^x. Use it for base-10 (order-of-magnitude) scales.
- [pow](/apl/scalar-functions/mathematical-functions/pow): Raises any base to a power. Use it when neither e, 2, nor 10 is the intended base.
- [log](/apl/scalar-functions/mathematical-functions/log): Returns the natural logarithm. Use it together with `exp` for natural-log-scale analysis.
