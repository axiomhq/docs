Use the `log10` function in APL to compute the common (base-10) logarithm of a positive numeric value. The function returns `null` for zero or negative inputs.

`log10` is useful for order-of-magnitude analysis. Because each unit increase in `log10(x)` represents a tenfold increase in `x`, you can use it to compare values that span multiple orders of magnitude, build decibel-scale metrics, or bucket data by powers of ten. It's also the basis for recovering values with `exp10`.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk SPL, `log10()` works identically: it returns the base-10 logarithm.

<CodeGroup>
```sql Splunk example
| eval log10_duration = log10(req_duration_ms)
````

```kusto APL equivalent
['sample-http-logs']
| extend log10_duration = log10(req_duration_ms)
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In ANSI SQL, `LOG10()` is a standard function in SQL Server and PostgreSQL with the same behavior as APL's `log10`.

<CodeGroup>
```sql SQL example
SELECT LOG10(req_duration_ms) AS log10_duration FROM logs
```

```kusto APL equivalent
['sample-http-logs']
| extend log10_duration = log10(req_duration_ms)
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
log10(x)
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `x` | real | Yes | A positive real number (x > 0). |

### Returns

- The base-10 logarithm of `x`.
- `null` if `x` is negative, zero, or cannot be converted to a real value.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `log10` to express request durations on a log scale where each integer step represents an order of magnitude, making it easy to identify very slow requests.

**Query**

```kusto
['sample-http-logs']
| where req_duration_ms > 0
| extend log10_duration = log10(req_duration_ms)
| project _time, id, req_duration_ms, log10_duration
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20req_duration_ms%20%3E%200%20%7C%20extend%20log10_duration%20%3D%20log10%28req_duration_ms%29%20%7C%20project%20_time%2C%20id%2C%20req_duration_ms%2C%20log10_duration%22%7D)

**Output**

| _time | id | req_duration_ms | log10_duration |
|---|---|---|---|
| 2024-11-14 10:00:00 | user-1 | 1.0 | 0.0000 |
| 2024-11-14 10:01:00 | user-2 | 100.0 | 2.0000 |
| 2024-11-14 10:03:00 | user-3 | 10000.0 | 4.0000 |

A request of 10 000 ms has a `log10_duration` of 4, while one of 1 ms has 0. Each integer difference represents a tenfold change in duration, making it easy to bucket into order-of-magnitude groups.

</Tab>
<Tab title="OpenTelemetry traces">

Use `log10` to compute the geometric mean of span durations per service in base-10 log space, then recover it using `exp10`.

**Query**

```kusto
['otel-demo-traces']
| extend duration_ms = duration / 1ms
| where duration_ms > 0
| summarize avg_log10 = avg(log10(duration_ms)) by ['service.name'], bin(_time, 1h)
| extend geometric_mean = exp10(avg_log10)
| project _time, ['service.name'], avg_log10, geometric_mean
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27otel-demo-traces%27%5D%20%7C%20extend%20duration_ms%20%3D%20duration%20%2F%201ms%20%7C%20where%20duration_ms%20%3E%200%20%7C%20summarize%20avg_log10%20%3D%20avg%28log10%28duration_ms%29%29%20by%20%5B%27service.name%27%5D%2C%20bin%28_time%2C%201h%29%20%7C%20extend%20geometric_mean%20%3D%20exp10%28avg_log10%29%20%7C%20project%20_time%2C%20%5B%27service.name%27%5D%2C%20avg_log10%2C%20geometric_mean%22%7D)

**Output**

| _time | service.name | avg_log10 | geometric_mean |
|---|---|---|---|
| 2024-11-14 10:00:00 | checkout | 2.448 | 280.5 |
| 2024-11-14 10:00:00 | cart | 2.147 | 140.2 |
| 2024-11-14 10:00:00 | frontend | 1.747 | 55.8 |

The `avg_log10` value of 2.448 for the checkout service means its geometric mean duration is 10^2.448 ≈ 280 ms.

</Tab>
<Tab title="Security logs">

Use `log10` to summarize error-response durations by order of magnitude per country, revealing which regions experience the most severe response-time degradation.

**Query**

```kusto
['sample-http-logs']
| where status == '500' and req_duration_ms > 0
| summarize avg_log10 = avg(log10(req_duration_ms)) by ['geo.country']
| extend avg_duration = exp10(avg_log10)
| project ['geo.country'], avg_log10, avg_duration
| order by avg_duration desc
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20status%20%3D%3D%20%27500%27%20and%20req_duration_ms%20%3E%200%20%7C%20summarize%20avg_log10%20%3D%20avg%28log10%28req_duration_ms%29%29%20by%20%5B%27geo.country%27%5D%20%7C%20extend%20avg_duration%20%3D%20exp10%28avg_log10%29%20%7C%20project%20%5B%27geo.country%27%5D%2C%20avg_log10%2C%20avg_duration%20%7C%20order%20by%20avg_duration%20desc%22%7D)

**Output**

| geo.country | avg_log10 | avg_duration |
|---|---|---|
| BR | 2.693 | 492.7 |
| DE | 2.302 | 200.3 |
| US | 2.107 | 128.0 |

Regions with the highest `avg_log10` consistently produce the slowest error responses. Each full unit of `avg_log10` represents a tenfold increase in average error latency.

## List of related functions

- [exp10](/apl/scalar-functions/mathematical-functions/exp10): Returns 10^x. Use it as the inverse of `log10` to recover original values.
- [log](/apl/scalar-functions/mathematical-functions/log): Returns the natural logarithm. Use it when you want the mathematically natural base rather than base-10.
- [log2](/apl/scalar-functions/mathematical-functions/log2): Returns the base-2 logarithm. Use it for binary-scale analysis.
- [pow](/apl/scalar-functions/mathematical-functions/pow): Raises a value to a power. Use it to compute powers of 10 directly as an alternative to `exp10`.
- [round](/apl/scalar-functions/mathematical-functions/round): Rounds a value. Use it after `log10` to bucket values by integer order of magnitude.
