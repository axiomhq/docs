title: asin

Use the `asin` function in APL to compute the arc sine (inverse sine) of a numeric expression. The function returns the angle, in radians, whose sine equals the input value. The input must be in the range [-1, 1]; values outside this range return `null`.

`asin` is useful when you work with normalized ratios or rates and need to map them to angular values. For example, you can use `asin` to encode success or error rates as phase angles for cyclic signal analysis.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk SPL, `asin()` is available in the `eval` command with the same behavior: it returns the arc sine in radians for an input in [-1, 1].

<CodeGroup>
```sql Splunk example
| eval angle = asin(normalized_rate)
````

```kusto APL equivalent
['sample-http-logs']
| extend angle = asin(normalized_rate)
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In ANSI SQL, `ASIN()` is a standard mathematical function. It accepts a value in [-1, 1] and returns the arc sine in radians.

<CodeGroup>
```sql SQL example
SELECT ASIN(normalized_rate) AS angle FROM logs
```

```kusto APL equivalent
['sample-http-logs']
| extend angle = asin(normalized_rate)
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
asin(x)
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `x` | real | Yes | A real number in the range [-1, 1]. |

### Returns

- The arc sine of `x` in radians, in the range [-π/2, π/2].
- `null` if `x` < -1 or `x` > 1.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `asin` to encode per-hour HTTP success rates as angular values. The resulting phase angle grows as the success rate increases from 0 to 1.

**Query**

```kusto
['sample-http-logs']
| summarize success = countif(status == '200'), total = count() by bin(_time, 1h)
| extend success_rate = todouble(success) / todouble(total)
| extend phase = asin(2 * success_rate - 1)
| project _time, success_rate, phase
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20summarize%20success%20%3D%20countif%28status%20%3D%3D%20%27200%27%29%2C%20total%20%3D%20count%28%29%20by%20bin%28_time%2C%201h%29%20%7C%20extend%20success_rate%20%3D%20todouble%28success%29%20%2F%20todouble%28total%29%20%7C%20extend%20phase%20%3D%20asin%282%20%2A%20success_rate%20-%201%29%20%7C%20project%20_time%2C%20success_rate%2C%20phase%22%7D)

**Output**

| _time | success_rate | phase |
|---|---|---|
| 2024-11-14 10:00:00 | 0.98 | 1.3694 |
| 2024-11-14 11:00:00 | 0.75 | 0.5236 |
| 2024-11-14 12:00:00 | 0.50 | 0.0000 |

The query maps `success_rate` from [0, 1] to [-1, 1] using `2 * success_rate - 1`, then computes `asin` to get a phase in [-π/2, π/2]. A phase near π/2 indicates near-100% success; a phase near -π/2 indicates near-0% success.

</Tab>
<Tab title="OpenTelemetry traces">

Use `asin` to encode per-service OK rates as angular values for anomaly detection across services.

**Query**

```kusto

| summarize ok = countif(status_code == '200'), total = count() by ['service.name'], bin(_time, 1h)
| extend ok_rate = todouble(ok) / todouble(total)
| extend phase = asin(2 * ok_rate - 1)
| project _time, ['service.name'], ok_rate, phase
```

<Tab title="Security logs">

Use `asin` to encode per-country HTTP error rates as phase angles, helping you compare the relative threat level across regions.

**Query**

```kusto
['sample-http-logs']
| summarize errors = countif(status == '500'), total = count() by ['geo.country'], bin(_time, 1h)
| extend error_rate = todouble(errors) / todouble(total)
| extend phase = asin(2 * error_rate - 1)
| project _time, ['geo.country'], error_rate, phase
```

## List of related functions

- [acos](/apl/scalar-functions/mathematical-functions/acos): Returns the arc cosine. Use it when the cosine encoding is more appropriate for your data.
- [atan](/apl/scalar-functions/mathematical-functions/atan): Returns the arc tangent. Use it when the input is not bounded to [-1, 1].
- [sin](/apl/scalar-functions/mathematical-functions/sin): Returns the sine. Use it to apply the forward transformation before using `asin` as the inverse.
- [radians](/apl/scalar-functions/mathematical-functions/radians): Converts degrees to radians. Use it if your angle data is in degrees before passing it to `asin`.
- [degrees](/apl/scalar-functions/mathematical-functions/degrees): Converts radians to degrees. Use it to convert the `asin` result into degrees.
