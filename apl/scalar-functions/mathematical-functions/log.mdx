Use the `log` function in APL to compute the natural logarithm (base-e) of a positive numeric value. The function is the inverse of `exp`.

`log` is one of the most commonly used mathematical functions in observability. Log-transforming latency, error counts, or request rates compresses wide value ranges into a more manageable scale, reduces the influence of extreme outliers, and can reveal patterns that are linear on a log scale. It's also the basis for computing geometric means with `exp(avg(log(x)))`.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk SPL, the natural logarithm is called `ln()` rather than `log()`. The SPL `log()` function computes the base-10 logarithm by default. In APL, `log()` always means the natural logarithm.

<CodeGroup>
```sql Splunk example
| eval ln_duration = ln(req_duration_ms)
````

```kusto APL equivalent
['sample-http-logs']
| extend ln_duration = log(req_duration_ms)
```
</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In ANSI SQL, `LN()` computes the natural logarithm and `LOG()` computes the base-10 logarithm in most dialects. In APL, `log()` means the natural logarithm, matching SQL's `LN()`.

<CodeGroup>
```sql SQL example
SELECT LN(req_duration_ms) AS ln_duration FROM logs
```

```kusto APL equivalent
['sample-http-logs']
| extend ln_duration = log(req_duration_ms)
```
</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
log(x)
```

### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `x` | real | Yes | A positive real number (x > 0). |

### Returns

- The natural logarithm of `x`.
- `null` if `x` is negative, zero, or cannot be converted to a real value.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Use `log` to compress request duration values from a wide scale onto a log scale, making it easier to spot patterns across orders of magnitude.

**Query**

```kusto
['sample-http-logs']
| where req_duration_ms > 0
| extend log_duration = log(req_duration_ms)
| project _time, id, req_duration_ms, log_duration
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20req_duration_ms%20%3E%200%20%7C%20extend%20log_duration%20%3D%20log%28req_duration_ms%29%20%7C%20project%20_time%2C%20id%2C%20req_duration_ms%2C%20log_duration%22%7D)

**Output**

| _time | id | req_duration_ms | log_duration |
|---|---|---|---|
| 2024-11-14 10:00:00 | user-1 | 1.0 | 0.0000 |
| 2024-11-14 10:01:00 | user-2 | 100.0 | 4.6052 |
| 2024-11-14 10:02:00 | user-3 | 10000.0 | 9.2103 |

A tenfold increase in duration produces an additive increment of about 2.303 on the log scale. This reveals proportional differences that are invisible on a linear scale.

</Tab>
<Tab title="OpenTelemetry traces">

Use `log` to compute the geometric mean of span durations per service, which is a better central-tendency measure for skewed latency distributions.

**Query**

```kusto
['otel-demo-traces']
| extend duration_ms = duration / 1ms
| where duration_ms > 0
| summarize geometric_mean = exp(avg(log(duration_ms))) by ['service.name'], bin(_time, 1h)
| project _time, ['service.name'], geometric_mean
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27otel-demo-traces%27%5D%20%7C%20extend%20duration_ms%20%3D%20duration%20%2F%201ms%20%7C%20where%20duration_ms%20%3E%200%20%7C%20summarize%20geometric_mean%20%3D%20exp%28avg%28log%28duration_ms%29%29%29%20by%20%5B%27service.name%27%5D%2C%20bin%28_time%2C%201h%29%20%7C%20project%20_time%2C%20%5B%27service.name%27%5D%2C%20geometric_mean%22%7D)

**Output**

| _time | service.name | geometric_mean |
|---|---|---|
| 2024-11-14 10:00:00 | checkout | 280.5 |
| 2024-11-14 10:00:00 | cart | 140.2 |
| 2024-11-14 10:00:00 | frontend | 55.8 |

`exp(avg(log(x)))` is the standard pattern for computing the geometric mean. It is more resistant to outlier spikes than the arithmetic mean.

</Tab>
<Tab title="Security logs">

Use `log` to build a log-scale summary of error-response durations per country and identify regions where errors cluster at unusually high latencies.

**Query**

```kusto
['sample-http-logs']
| where status == '500' and req_duration_ms > 0
| summarize avg_log_duration = avg(log(req_duration_ms)) by ['geo.country']
| extend avg_duration = exp(avg_log_duration)
| project ['geo.country'], avg_log_duration, avg_duration
| order by avg_duration desc
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B%27sample-http-logs%27%5D%20%7C%20where%20status%20%3D%3D%20%27500%27%20and%20req_duration_ms%20%3E%200%20%7C%20summarize%20avg_log_duration%20%3D%20avg%28log%28req_duration_ms%29%29%20by%20%5B%27geo.country%27%5D%20%7C%20extend%20avg_duration%20%3D%20exp%28avg_log_duration%29%20%7C%20project%20%5B%27geo.country%27%5D%2C%20avg_log_duration%2C%20avg_duration%20%7C%20order%20by%20avg_duration%20desc%22%7D)

**Output**

| geo.country | avg_log_duration | avg_duration |
|---|---|---|
| BR | 6.20 | 492.7 |
| DE | 5.30 | 200.3 |
| US | 4.85 | 128.0 |

Regions with the highest `avg_duration` (after reversing the log with `exp`) experience the slowest error responses, which may indicate infrastructure problems or targeted attacks.

## List of related functions

- [exp](/apl/scalar-functions/mathematical-functions/exp): Returns e^x. Use it as the inverse of `log` to return to the original scale.
- [log2](/apl/scalar-functions/mathematical-functions/log2): Returns the base-2 logarithm. Use it for binary-scale analysis such as bit depth or memory sizing.
- [log10](/apl/scalar-functions/mathematical-functions/log10): Returns the base-10 logarithm. Use it for order-of-magnitude analysis or decibel calculations.
- [loggamma](/apl/scalar-functions/mathematical-functions/loggamma): Returns the log of the absolute value of the gamma function. Use it to avoid overflow when computing `gamma` on large inputs.
- [sqrt](/apl/scalar-functions/mathematical-functions/sqrt): Returns the square root. Use it as a lighter alternative to `log` for compressing small value ranges.
