Use the `tolong` function to convert various data types to a long (signed 64-bit) integer value. This is helpful when you need to normalize numeric values from different sources into long integer format for mathematical operations, comparisons, or when working with large numeric identifiers.

You typically use `tolong` when working with numeric strings, floating-point numbers, or other types that need to be converted to long integers, especially when dealing with large numbers that exceed the standard integer range.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk, you use `tonumber` to convert values to numbers. In APL, `tolong` specifically converts to long integers, which can handle larger values than standard integers.

<CodeGroup>
```sql Splunk example
... | eval timestamp_ms = tonumber(timestamp_string)
```

```kusto APL equivalent
... | extend timestamp_ms = tolong(timestamp_string)
```

</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In standard SQL, you use `CAST(... AS BIGINT)` or `CAST(... AS LONG)` to convert values to long integers. In APL, `tolong` provides a simpler way to convert to long integer values.

<CodeGroup>
```sql SQL example
SELECT CAST('241' AS BIGINT) AS id FROM logs;
```

```kusto APL equivalent
['sample-http-logs']
| extend id = tolong('241')
```

</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
tolong(value)
```

### Parameters

| Name  | Type    | Description                        |
| ----- | ------- | ---------------------------------- |
| value | dynamic | The value to convert to long. |

### Returns

If conversion is successful, the result is a long number. If conversion isn't successful, the result is `null`.

## Use case examples

<Tabs>
<Tab title="Log analysis">

Convert string representations of large numeric identifiers to long integers for numeric operations and comparisons.

**Query**

```kusto
['sample-http-logs']
| extend user_id_long = tolong(['id'])
| extend is_large_id = user_id_long > 1000000
| where is_large_id == true
| project _time, ['uri'], ['id'], user_id_long, is_large_id
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B'sample-http-logs'%5D%20%7C%20extend%20user_id_long%20%3D%20tolong(%5B'id'%5D)%20%7C%20extend%20is_large_id%20%3D%20user_id_long%20%3E%201000000%20%7C%20where%20is_large_id%20%3D%3D%20true%20%7C%20project%20_time%2C%20%5B'uri'%5D%2C%20%5B'id'%5D%2C%20user_id_long%2C%20is_large_id%22%7D)

| ----- | --- | -- | ------------ | ----------- |
| Jun 24, 09:28:10 | /api/users | 1234567 | 1234567 | true |

This example converts string user IDs to long integers, enabling numeric comparisons to identify users with large identifier values.

## List of related functions

- [toint](/apl/scalar-functions/conversion-functions/toint): Converts input to integer. Use `toint` for standard integer ranges, and `tolong` for larger values.
- [todouble](/apl/scalar-functions/conversion-functions/todouble): Converts input to real number. Use `todouble` when you need decimal precision, and `tolong` when you need whole numbers.
- [tohex](/apl/scalar-functions/conversion-functions/tohex): Converts integer to hexadecimal string. Use `tohex` with `tolong` to convert large integers to hex format.
