Use the `tolong` function to convert various data types to a long (signed 64-bit) integer value. This is helpful when you need to normalize numeric values from different sources into long integer format for mathematical operations, comparisons, or when working with large numeric identifiers.

You typically use `tolong` when working with numeric strings, floating-point numbers, or other types that need to be converted to long integers, especially when dealing with large numbers that exceed the standard integer range.

## For users of other query languages

If you come from other query languages, this section explains how to adjust your existing queries to achieve the same results in APL.

<AccordionGroup>
<Accordion title="Splunk SPL users">

In Splunk, you use `tonumber` to convert values to numbers. In APL, `tolong` specifically converts to long integers, which can handle larger values than standard integers.

<CodeGroup>
```sql Splunk example
... | eval timestamp_ms = tonumber(timestamp_string)
```

```kusto APL equivalent
... | extend timestamp_ms = tolong(timestamp_string)
```

</CodeGroup>

</Accordion>
<Accordion title="ANSI SQL users">

In standard SQL, you use `CAST(... AS BIGINT)` or `CAST(... AS LONG)` to convert values to long integers. In APL, `tolong` provides a simpler way to convert to long integer values.

<CodeGroup>
```sql SQL example
SELECT CAST('241' AS BIGINT) AS id FROM logs;
```

```kusto APL equivalent
['sample-http-logs']
| extend id = tolong('241')
```

</CodeGroup>

</Accordion>
</AccordionGroup>

## Usage

### Syntax

```kusto
tolong(value)
```

### Parameters

| Name  | Type    | Description                        |
| ----- | ------- | ---------------------------------- |
| value | dynamic | The value to convert to long. |

### Returns

If conversion is successful, the result is a long number. If conversion isn't successful, the result is `null`.

## Use case example

Convert string representations of HTTP status codes to long integers for numeric operations and comparisons.

**Query**

```kusto
['sample-http-logs']
| extend status_long = tolong(status)
| extend is_error = status_long >= 400
| where is_error == true
| project _time, uri, status, status_long, is_error
```

[Run in Playground](https://play.axiom.co/axiom-play-qf1k/query?initForm=%7B%22apl%22%3A%22%5B'sample-http-logs'%5D%20%7C%20extend%20status_long%20%3D%20tolong(status)%20%7C%20extend%20is_error%20%3D%20status_long%20%3E%3D%20400%20%7C%20where%20is_error%20%3D%3D%20true%20%7C%20project%20_time%2C%20uri%2C%20status%2C%20status_long%2C%20is_error%22%7D)

**Output**

| _time | uri | status | status_long | is_error |
| ----- | --- | -- | ------------ | ----------- |
| Jun 24, 09:28:10 | /api/users | 500 | 500 | true |

This example converts string HTTP status codes to long integers, enabling numeric comparisons to identify error responses.
