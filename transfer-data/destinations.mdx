To transfer and transform data from an Axiom dataset to a destination, you need to set up a destination. This is where data is transferred. Once you set up a destination, you can use it in any flow.

To set up a destination:
1. Click the **Flows** tab. Axiom displays the list of flows you have created.
2. In the top right, click **Manage destinations**, and then click **New destination**.
3. Name the destination.
4. In **Kind**, select the destination type.
5. Configure the destination. For more information on each destination type, see the following:

    - [AWS S3](#configure-aws-s3-destination)
    - [Axiom](#configure-axiom-dataset-destination)
    - [Azure](#configure-azure-destination)
    - [Elastic Bulk](#configure-elastic-bulk-destination)
    - [HTTP](#configure-http-destination)
    - [HTTP (Authorization)](#configure-http-authorization-destination)
    - [HTTP (Basic)](#configure-http-basic-destination)
    - [OpenTelemetry Traces](#configure-opentelemetry-traces-destination)
    - [Chronicle](#configure-chronicle-destination)
    - [Cribl](#configure-cribl-destination)

6. In the bottom right, click **Save**.

## Configure AWS S3 destination

Configure the following:

- **Access key ID**
- **Secret access key**
- **Region**
- **Bucket**

## Configure Axiom dataset destination

1. Create a destination dataset in Axiom where you want to transfer the data.
2. Create an Axiom API token with permissions to update the destination dataset.
3. Configure the following:

    - In **Dataset**, specify the name of the destination dataset.
    - In **Token**, specify the Axiom API token.
    - In **URL**, specify `https://api.axiom.co/`.

## Configure Azure destination

Configure the following:

- **URL**
- **Format**
- **Tenant ID**
- **Client ID**
- **Client Secret**
