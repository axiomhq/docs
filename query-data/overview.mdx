Console is the main interface for interacting with Axiom. It provides a set of tools for querying, visualizing, and managing your data.

When you open Console, you see the following tabs for interacting with your data:

- [**Query**](/query-data/explore): Run queries against your data using either a visual interface or a text-based query language.
- [**Stream**](/query-data/stream): Watch events arrive in real time and inspect individual events as they're ingested.
- [**Monitors**](/monitor-data/monitors): Create and manage monitors to alert you when your data meets certain criteria.
- [**Dashboards**](/dashboards/create): Create and manage dashboards to visualize your data.
- [**Datasets**](/query-data/datasets): Browse and analyze the fields in any dataset. Use this to understand the shape of your data before writing a query.
- [**AI engineering**](/console/intelligence/ai-agents-overview): Use AI agents to query your data and build dashboards.

## Query data

Axiom provides powerful tools to query your data. Whether you work with logs, traces, events, or metrics, and whether you prefer a visual interface or a text-based query language, Axiom has a solution for you.

The Query tab is where you run queries against your data. It offers two interfaces:

- [**Builder**](/query-data/explore): A visual interface for filtering, grouping, and visualizing data.
- [**Editor**](/query-data/query-editor): Write queries in a text-based query language.

Within Editor, you can write queries in two different languages, each designed for different data types:

- [**APL**](/apl/introduction) (Axiom Processing Language) queries logs, traces, and events stored in [EventDB](/platform-overview/architecture).
- [**MPL**](/mpl/introduction) (Metrics Processing Language) queries metrics stored in [MetricsDB](/platform-overview/architecture).

### Tools to help you build queries

You can use the following features for all machine data types:

- [Canvas](/query-data/canvas): A lightweight workspace to display multiple charts side by side for quick visual correlation and comparison.
- [Saved queries](/query-data/datasets#saved-queries): Save and share queries with team members for easy reuse.

The following features are currently only available for logs, traces, and events:

- [Views](/query-data/views): Apply filters and transformations to datasets to create reusable virtual tables with scoped access control.
- [Virtual fields](/query-data/virtual-fields): Derive new values from data in real-time during queries without pre-structuring or transforming data during ingestion.
- [Generate query using natural language](/query-data/query-editor#generate-query-using-natural-language): Use natural language to generate APL queries in Console.

### Other ways to query data

In addition to Console, you can also query data in the following ways:

| Method | Description |
|:---|:---|
| [MCP Server](/console/intelligence/mcp-server) | A Model Context Protocol (MCP) server that enables AI agents to query your data. |
| [Skills](/console/intelligence/skills) | Give AI agents the ability to query your data. |
| [API](/restapi/query) | Query data programmatically using the Axiom API. |

AI agents can use MCP Server and Skills for many other tasks like building dashboards. For more information, see [AI agents](/console/intelligence/ai-agents-overview).

## Dashboards

Dashboards bring together visualizations of queries across multiple datasets in one place. You can add time series charts, tables, statistics, heatmaps, and other [dashboard elements](/dashboard-elements/create) to build a shared view of your data.

Dashboards support [filters](/query-data/filters) that apply across all elements at once and [annotations](/query-data/annotate-charts) to mark deployments or incidents on your charts.

For more information, see [Dashboards](/dashboards/create).

## Monitors

Monitors are background tasks that periodically run queries you define. When results meet the conditions you set, Axiom sends you an alert through a [notifier](/monitor-data/notifiers-overview) of your choice, such as email, Slack, PagerDuty, or a custom webhook.

Axiom supports three types of monitor:

- [Threshold monitors](/monitor-data/threshold-monitors): Alert when an aggregated value crosses a limit you set.
- [Match monitors](/monitor-data/match-monitors): Alert when events match a specific filter.
- [Anomaly monitors](/monitor-data/anomaly-monitors): Alert when values deviate from expected behavior based on historical data.

For more information, see [Monitors](/monitor-data/monitors).

## Manage your data

You can manage your data in the following ways:

- [Datasets](/query-data/datasets): Browse and analyze the fields in any dataset. Use this to understand the shape of your data before writing a query.
- [Views](/query-data/views): Apply filters and transformations to datasets to create reusable virtual tables with scoped access control.
- [Virtual fields](/query-data/virtual-fields): Derive new values from data in real-time during queries without pre-structuring or transforming data during ingestion.
- [Access control](/reference/settings): Manage users and their permissions.
- [Tokens](/reference/tokens): Manage API keys for programmatic access to Axiom.
- [Audit log](/reference/audit-log): Track every change and action taken by users within Axiom.
- [Organization settings](/reference/organization-settings): Manage your organization's settings.
- [Usage billing](/reference/usage-billing): View your organization's usage and billing.
- [Profile](/reference/profile): View your profile and settings.

## Extend

Extensions augment your Axiom experience by connecting to third-party platforms and cloud services. Use them to set up prebuilt dashboards, configure log forwarding, or connect tools like [Grafana](/apps/grafana), [Vercel](/apps/vercel), and [Terraform](/apps/terraform), [AWS Lambda](/apps/lambda), [CloudFlare Workers](/apps/cloudflare-workers), and [Netlify](/apps/netlify) to your Axiom organization.

For more information, see [Extensions](/apps/introduction).

## Intelligence

Axiom's intelligence features help you reach insights faster by combining the data platform with machine-assisted analysis.

- [Spotlight](/console/intelligence/spotlight): Automatically compare a subset of events against a baseline to surface the most significant differences across every field.
- [AI-assisted workflows](/console/intelligence): Use AI to help you build queries and dashboards.
- [AI agents](/console/intelligence/ai-agents-overview): Connect AI agents to Axiom through [MCP Server](/console/intelligence/mcp-server) or [Skills](/console/intelligence/skills) to query data, build dashboards, and automate analysis.
