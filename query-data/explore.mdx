Axiom provides powerful tools to query your data. Whether you work with logs, traces, events, or metrics, and whether you prefer a visual interface or a text-based query language, Axiom has you covered.

Console has three tabs for exploring your data:
- [**Datasets**](/query-data/datasets): Browse and analyze the fields in any dataset. Use this to understand the shape of your data before writing a query.
- [**Stream**](/query-data/stream): Watch events arrive in real time and inspect individual events as they're ingested.
- **Query**: Run queries against your data using either a visual interface or a text-based query language.

The Datasets and Stream tabs work the same way for all data types. The Query tab experience differs based on what you query and how you query it.

## Query tab

The Query tab is where you run queries against your data. It offers two interfaces:

- **Builder**: A visual interface for filtering, grouping, and visualizing data.
- **Editor**: Write queries in a text-based query language.

Axiom has two query languages, each designed for different data types:

- **APL** (Axiom Processing Language) queries logs, traces, and events stored in [EventDB](/platform-overview/architecture).
- **MPL** (Metrics Processing Language) queries metrics stored in [MetricsDB](/platform-overview/architecture).

Both Builder and Editor are available for each query language.

| | Builder (visual) | Editor (query language) |
|:---|:---|:---|
| Logs, traces, events | [Query using Builder](/query-data/explore#create-query-using-visual-query-builder)<br />[Traces](/query-data/traces) for trace-specific UI | [Query using APL](/query-data/explore#create-query-using-apl)<br />[APL reference](/apl/introduction) |
| Metrics | [Query using Builder](/query-data/metrics/query-metrics) | [Query using MPL](/query-data/metrics/query-metrics-mpl) |

The following features are available for logs, traces, and events:

- [Views](/query-data/views): Apply filters and transformations to datasets to create reusable virtual tables with scoped access control.
- [Virtual fields](/query-data/virtual-fields): Derive new values from data in real-time during queries without pre-structuring or transforming data during ingestion.

## Query data with AI

Console supports multiple ways to query data with AI:

| Method | Description | Query language |
|:---|:---|:---|
| [Query using natural language](/query-data/query-natural-language) | Use natural language to generate APL queries in Console. | APL |
| [MCP Server](/console/intelligence/mcp-server) | A Model Context Protocol (MCP) server that enables AI agents to query your data. | APL, MPL |
| [Skills](/console/intelligence/skills) | Give AI agents the ability to query your data. | APL, MPL |

AI agents can use MCP Server and Skills for many other tasks like building dashboards. For more information, see [AI agents](/console/intelligence/ai-agents-overview).

## Query data with API

You can query data programmatically using the Axiom API. For more information, see [Query data via Axiom API](/restapi/query).
